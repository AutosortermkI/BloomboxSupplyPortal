# BloomBox Supply Portal

Local supplier dashboard plus scraper tooling for BloomBox / Second Chance
Plants procurement research.

## Current Reality

- `index.html` is a static browser dashboard. It can be opened directly from
  disk and does not require a web server.
- The dashboard is not a hosted application, does not include authentication,
  and does not run scraper jobs by itself.
- The live feed shown in the Supplier Portal tab comes from root-level
  `prices.js`, which is generated by `python -m scrape.run`.
- The browser no longer stores supplier passwords. It stores only non-secret
  login metadata; scraper passwords belong in the local OS credential store.
- As of May 6, 2026, the dashboard is sourced from
  `data/master/suppliers.canonical.json`, not the old raw supplier array.
  The current canonical registry contains 14 verified suppliers, 111 explicit
  review-queue rows, and 1 rejected duplicate raw row.
- `data/raw/current_dashboard_suppliers.*` preserves the old dashboard import
  as untrusted raw inventory only. Do not use it as identity proof.
- `data/master/adapter_supplier_map.csv` is the adapter source of truth:
  17 registered adapters are either mapped to one canonical supplier or
  quarantined with a reason.

## Local Use

Open the dashboard:

```bash
open index.html
```

Run the scraper preflight from the repo root:

```bash
python3 -m scrape.run --preflight
```

Run the registered public adapters:

```bash
python3 -m scrape.run --public -v
```

After a successful run, the dashboard reads:

- `prices.js` at the repo root
- `scrape/output/prices.json`
- `scrape/output/prices.js`

## Supplier Registry

The supplier list is rebuilt through a reproducible verification pipeline. The
raw import was extracted from the pre-canonical dashboard. After the dashboard
rewrite, `index.html` intentionally contains only verified canonical rows, so
rerun extraction only against the actual pre-canonical dashboard source file
being audited.

```bash
python3 tools/audit_supplier_registry.py
python3 tools/audit_supplier_registry.py --canonical data/master/suppliers.canonical.json
```

Registry artifacts:

- `data/raw/current_dashboard_suppliers.json` and `.csv`: raw dashboard import,
  unchanged except for serialization.
- `reports/supplier_master_audit.md`: duplicate IDs, duplicate domains, adapter
  mismatches, placeholders, and missing-field audit.
- `data/master/suppliers.canonical.json` and `.csv`: verified suppliers only;
  every verified record has evidence URLs.
- `data/master/supplier_review_queue.csv`: uncertain suppliers that must not be
  promoted silently.
- `data/master/supplier_rejections.csv`: rejected or merged raw records with
  explicit reasons.
- `data/master/adapter_supplier_map.csv`: registered adapter to canonical
  supplier mapping or quarantine reason.

## Credentials

Credentialed supplier scraping is local-runtime only. Use:

```bash
python3 -m scrape.credentials set --id 110 --url https://www.waltersgardens.com/customer/account/login/ --user "$BLOOMBOX_WALTERS_USERNAME"
```

The command prompts for the supplier password, stores the password in macOS
Keychain, and writes non-secret metadata to:

```text
~/.config/bloombox/credentials.json
```

Legacy plaintext dashboard vault exports are not auto-discovered. They are
accepted only when explicitly passed via `BLOOMBOX_VAULT_PATH` or
`load_vault(path=...)` for migration/debugging.

Inspect configured runtime credentials without printing passwords:

```bash
python3 -m scrape.credentials list
```

American Native Plants is configured locally and verified through the runner:

```bash
python3 -m scrape.run --id 43 --concurrency 1 -v
```

The May 6, 2026 live run extracted 462 rows from the official April 30, 2026
availability workbook. The supplier login page is protected by reCAPTCHA, so
the adapter uses the official public workbook for availability and does not
emit manufactured prices.

## Deployment Model

The current defensible model is:

- local static dashboard for internal procurement review
- local scraper runner that writes `prices.js`
- optional scheduled local job once adapter quality is known

A hosted static dashboard is possible later, but it should only host static
dashboard assets and non-secret feed artifacts. Supplier credentials and
scraper execution should remain in a separate private runner environment.

## Project Structure

```text
BloomboxSupplyPortal/
  index.html              static dashboard
  prices.js               generated live-feed script when scraper publishes
  data/
    raw/                  untrusted dashboard import artifacts
    master/               verified registry, review queue, rejections, adapter map
  reports/                supplier registry audit reports
  scrape/
    run.py                scraper CLI and feed publisher
    credentials.py        local credential helper
    ADAPTER_AUDIT.md      current adapter quality classifications
    core/                 fetcher, adapter, vault, extraction primitives
    adapters/             supplier-specific adapters
    output/               generated feed, summary, and history files
  tools/                  registry extraction and audit utilities
  tests/                  scraper/runtime and registry regression tests
```
